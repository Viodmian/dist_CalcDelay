# DelayScope 项目宣传文案

## 长文版：让多通道音频时延看得见

在多通道采集、麦克风阵列、回声链路分析、设备联调和音频算法验证里，一个很常见但又很难快速定位的问题是：不同通道之间到底差了多少时间？

有时候这个差异只有几十个采样点，听感上不一定明显，但它会直接影响后续的波束形成、AEC、定位、同步校准和链路诊断。更麻烦的是，时延不一定是稳定的。某些文件里可能夹杂静音、噪声、截幅、填充、丢帧或异常片段，单次计算出来的结果并不一定可信。

**DelayScope** 就是为这个场景做的一个轻量级桌面工具。

它可以读取多通道 WAV / PCM 音频，以指定通道作为参考，计算其它通道相对参考通道的 `delay_samples` 和毫秒级时间差。工具内部使用 GCC-PHAT 做时延估计，并采用多段统计方式，把长音频拆成多个片段分别分析，再用中位数汇总最终结果，让结果对异常片段更稳健。

DelayScope 不只给一个最终数字。它还会展示每个分段的 Delay Drift 轨迹，让你能直观看到时延是否稳定、是否存在缓慢漂移或突发跳变。同时，它会计算归一化相关系数 `|rho|` 作为置信度参考，帮助判断某个通道或某个片段的结果是否值得相信。

除了时延本身，DelayScope 还内置了一些音频质量辅助判断：宽带 / 窄带判定、数字截幅检测、疑似填充或丢帧统计。这些信息可以帮助你快速区分“算法估计问题”和“输入数据质量问题”，减少排查时间。

### 它适合谁

- 做多麦克风阵列、AEC、音频同步或声学链路分析的工程师
- 需要检查采集卡、DSP、设备通道之间固定延迟的人
- 需要快速判断多通道录音是否存在漂移、错位、截幅或异常片段的人
- 想把原本靠脚本和日志判断的问题，变成可视化结果的人

### 它解决什么问题

DelayScope 的目标不是做一个复杂的音频工作站，而是把“多通道时延分析”这件事做得直接、清楚、可复查。

你可以快速看到：

- 每个通道相对参考通道提前还是滞后
- 时延对应多少毫秒
- 这个时延在整段音频里是否稳定
- 哪些片段置信度高，哪些片段不可信
- 输入数据是否存在截幅、填充、丢帧或频带异常

### 项目亮点

- **多通道支持**：支持 WAV 自动识别采样率和通道数，也支持手动指定 PCM 参数。
- **GCC-PHAT 时延估计**：适合语音、多路径和增益差异场景下的通道相对时延分析。
- **多段统计**：分段计算后取中位数，减少异常片段对最终结果的影响。
- **Delay Drift 可视化**：通过轨迹图观察时延是否稳定，方便排查采样率偏差、时钟漂移和链路异常。
- **置信度分析**：用 `|rho|` 辅助判断结果可靠性，不只给数字，也给判断依据。
- **音频质量诊断**：提供宽带 / 窄带、数字截幅、疑似填充或丢帧等辅助信息。
- **桌面化界面**：不需要每次写脚本、看数组、翻日志，打开工具就能完成分析。

### 一句话介绍

**DelayScope 是一个面向多通道音频工程分析的桌面工具，用于快速估计通道间时延、观察时延漂移，并判断结果可靠性和输入音频质量。**

如果你正在处理多通道音频同步、麦克风阵列、回声链路、采集设备校准或算法前处理，DelayScope 可以把原本分散在脚本、日志和波形里的信息集中展示出来，让通道时延问题更容易被发现、解释和复现。

## GitHub 项目简介

DelayScope is a desktop tool for multi-channel audio delay analysis. It estimates channel-to-reference delay with GCC-PHAT, summarizes results across segments, visualizes delay drift, reports confidence, and adds practical audio quality checks such as bandwidth labeling and clipping/fill detection.
